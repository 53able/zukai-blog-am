AI Agent History · 2022-2026

AIエージェントは、
爆死して実用化した。

2022年の原型、2023年の熱狂、2023〜2024年の幻滅、そして2024年後半からの実用化を、技術と設計思想の転換点として読む。

4年間の流れ

年表の主役は出来事そのものではなく、全自動への期待が、失敗を経て実用的な分業設計へ変わったこと。

2022

原型が生まれる

MRKL が道具を使う原型を、ReAct が考えてから動く原型を示した。

MRKL ReAct
2023

熱狂する

AutoGPT と BabyAGI が、自律的に動くソフトウェアの光景を作り手に見せた。

AutoGPT BabyAGI
2023〜2024

つまずく

無限ループ、ハルシネーション、課金増大、目的逸脱で、全自動丸投げの限界が見えた。

無限ループ 人間チェック不足
2024後半〜2025

実用化へ進む

領域特化エージェントが、コーディング、ブラウザ操作、リサーチの実務に乗り始めた。

Devin Claude Code

3つの転換点

この記事の面白さは、ツールや製品名の列挙ではなく、AIエージェントをどう扱うべきかの思想が変わったところにある。

転換点 1

原型が揃った

Tool Use と推論行動ループが組み合わさり、「考えて道具を使う」ソフトウェアの形が見えた。

転換点 2

全自動幻想が崩れた

タスクだけ渡せば全部やる、という設計では暴走や失敗に気づけず、実務に乗らなかった。

転換点 3

実用化の条件が揃った

モデル、文脈長、ツール規格、設計思想が揃い、領域特化エージェントが使える形に近づいた。

実用化を支えた4要因

2024年後半からの変化は、単一のブレイクスルーではなく、複数の成熟が同時に進んだ結果として整理されている。

1

長いタスクを扱えるモデル

AIが自力でこなせるタスクの長さが伸び、複数手順の仕事を任せやすくなった。

2

大きなコンテキスト

途中経過やツール結果を抱えたまま、文脈を失わずに進める余地が広がった。

3

ツール利用の規格化

Function Calling や MCP により、外部ツールやデータとの接続が安定した。

4

設計思想の成熟

全自動ではなく、人間の確認点と分業を前提にした実装へ変わった。

AIエージェントは、
名前ではなく機能になる。

あらゆるソフトウェアがエージェント化すると、「AIエージェント」という言葉自体が特別ではなくなるかもしれない。
