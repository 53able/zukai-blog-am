Claude Code / Codex Hooks / nvim --server

Agent followは、
Neovimを追従させる。

AI Agent の tool 実行を hook で拾い、helper 経由で nvim --server に渡すと、tmux 上の Neovim が編集箇所を追いかけます。

Source: Algomatic Tech Blog「Claude Code / Codex Hooks と nvim --server で作る terminal-native IDE」をもとに再構成。

Agent が編集しても、エディタは置いていかれる。

tmux の左右に Neovim と Agent を並べても、Agent が次々ファイルを触ると、人間は「今どこを直しているか」を見失います。

Before

頼めば開く、は忘れられる。

「編集したら Neovim で開いて」と毎回頼む運用は、コンテキストが長くなるほど抜け落ちます。

Agent の作業位置が見えない
手動確認で流れが止まる
頼みごと自体がトークンを使う
After

hook が拾えば、自動で追える。

tool 実行の前後で hook を発火させ、helper から Neovim の socket を叩きます。

編集前に対象行へジャンプ
編集後にファイルを再オープン
必要なら Agent が Neovim を読む

hook、helper、socket。配管は薄く分ける。

Agent ごとの差分は入口で吸収し、最後は同じ helper から同じ Neovim socket に集約します。

tmux window 単位で対応する作業面

AI Agent

Claude Code / Codex が Edit、MultiEdit、Write などの tool を実行します。

PreToolUse: これから編集する
PostToolUse: 編集が終わった
hook tool_input から file_path と old_string を取り出す
helper Agent ごとの payload や patch 形式を解釈する
socket repo と tmux window に対応する Neovim へ接続する

Neovim

nvim --server が外部からの open、jump、reload を受け取り、画面を Agent の編集位置へ追従させます。

ファイルを開く
該当行へ移動する
buffer / diagnostics を返す
実線: 編集追従 点線: 逆方向読取 薄線: ON/OFF 確認

追従は、3つの小さな動きでできている。

大きな IDE を作る話ではありません。tool の前後に小さく反応し、同じ socket で逆方向にも広げます。

01

編集前ジャンプ

PreToolUse で old_string の先頭行を探し、変更される前の場所へカーソルを動かします。

対象: Edit / MultiEdit
入力: file_path / old_string
02

編集後オープン

PostToolUse で変更後のファイルを開き、Neovim 側の buffer を最新状態へ寄せます。

対象: Edit / MultiEdit / Write
入力: file_path
03

逆方向に読む

同じ socket から buffer、cursor、LSP diagnostics を取り、Agent が Neovim の状態を見ます。

用途: diagnostics 確認
用途: Neovim 側の挙動調査

socket は、作業単位で一意にする。

固定名の socket では、別プロジェクトや別 tmux window の Neovim に誤接続します。だから名前に接続先の文脈を入れます。

repo name

プロジェクトを識別する。

開いているリポジトリ名を socket 名に含め、違うプロジェクトの Neovim へ飛ばないようにします。

nvim_<repo-name>
repo root hash

同名 repo の衝突を避ける。

同じ名前の repo が別パスにある場合でも、root path の hash で接続先を分けます。

_<repo-root-hash>
tmux window id

window ごとに対応させる。

同じプロジェクトを複数 window で開いても、Agent と Neovim の対応が混ざりません。

_<tmux-window-id>.sock
Takeaway

Agent follow は、terminal-native IDE の小さな配管。

hook は検知、helper は翻訳、socket は接点。入口だけ Agent ごとに変えれば、Neovim への追従は同じ仕組みに集約できます。
