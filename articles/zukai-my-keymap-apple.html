図解構成 / Personal Keymap Design

指の仕事を、
身体に合わせて配り直す。

小指に集まりすぎた負担を親指へ。遠い数字と矢印を f / j の触覚基準へ。

Source: Algomatic Tech Blog「こだわりキーマップ」をもとに再構成。

標準配列は、指の仕事量が不公平。

毎日何時間も触る道具だから、少しの負担も年単位で蓄積します。出発点は「どの指に、どの仕事を任せるべきか」です。

小

小指が忙しい。

Shift、Ctrl、Enter、BS。押しっぱなしや遠いキーで負担が集中します。

f/j

数字と矢印が遠い。

ホームポジションから離れるため、視線が落ち、ブラインドタッチしにくくなります。

number rowarrow keysPgUp / PgDn
Design Principle
器用な指に仕事を寄せ、
不器用な指から仕事を奪う。
親指高頻度キーを担当。BSとShiftを近くへ。
小指同時押しから解放。押しっぱなしの痛みを減らす。
f / j突起を触覚の支点に。数字と矢印を手元へ。

二層構成で、身体に合わせる。

macOS側のKarabinerでベースを組み替え、HHKB Hybrid本体のFnレイヤーで第二レイヤーを作ります。

HHKB Hybrid

本体側のFnレイヤー。

Fnを押している間だけ、遠い数字・矢印・記号をホームポジション周辺へ下ろします。

数字 → 左手 f 付近
矢印 → 右手 j / IJKL 付近
PgUp / PgDn / Home → 同レイヤー
各種記号 → 手元で完結

キーの置換ではなく、狙いの置換。

重要なのは「どのキーをどこへ動かしたか」より、「どの負担をどの指に移したか」です。

Conclusion

キーマップは、身体に合わせて設計できる。

小指に偏った負担を親指へ移し、f / j を支点に数字と矢印を手元へ下ろす。ほんの数キーの再配置でも、毎日の負担と速度は変わります。
